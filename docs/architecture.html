Bollydog Framework Architecture

Async microservice framework built on mode — Commands as executable units, Hub as central dispatcher, Protocols as pluggable data layers.

Design Rules Constraints

#Rule
1Command = thin orchestration, no owned state
2AppService = resource owner, expose business methods
3Protocol = environment abstraction (swap for tests)
4Use globals.app — never reach into sub-services
5Cross-domain: hub.dispatch or yield cmd only
6AppService exposes capabilities, Commands schedule
